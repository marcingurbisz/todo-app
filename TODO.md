Tap folders to expand, tap files to open the editor, and use **Move** to see how a file flows between directories. A few design notes on what's prototyped and what the choices imply:

**The tree is the home screen.** The top-level folders in sort order are `__now` → `__today` → `_short-term` → `review-*` — that matches the semantic priority of your underscore convention and keeps the ones you touch most at the top. I used small colored badges (coral for `__now`, amber for `__today`, blue for `_short-term`, purple for the review cycles, gray for `reviewed/`) so you can locate a status at a glance without reading the path.

The individual `⋯` on each file opens the same move sheet with context-aware "suggested" destinations at the top — so moving `publishing.md` from `__today` surfaces `__today/tomorrow`, `_short-term`, and `__now` as the three likely next statuses.
Every directory row now has its own `⋯` which opens a sheet with *New file here* and, when applicable, *Move all N files up to {parent}/*. The "move up one level" framing you described is now the general mechanism — `tomorrow → __today` surfaces it as the daily morning gesture, but it's the same action shape everywhere, so you're not locked into predefined workflows.

**Editor with Preview / Raw toggle.** Preview renders real markdown via marked; Raw is a monospace textarea. Tapping the preview body switches to Raw with the draft preserved — so "editable preview" in this prototype means the preview *invites* editing by becoming the editor on tap. A true inline WYSIWYG (contenteditable with markdown-aware per-line styling, iA Writer–style) would be the natural next step if you want to avoid the mode switch entirely.

**Sync is a silent commit.** Every action (save, move, delete, create) triggers the `publishing…` state in the top-right chip followed by a toast — no save dialogs, no commit messages. This matches your TODO note that concurrency is low and conflicts should be rare; the app just commits and pushes, and any rare conflict would surface as a retry via "refresh and try again" per your `concurrency-guidance.md`.

**What's deliberately missing** and worth a follow-up decision:
- Search — once you have 40+ files this becomes non-negotiable.
- Conflict UI — right now the prototype just assumes success. A "remote moved ahead, refresh?" banner on the editor when push fails would be the minimal real-world version.

The `+` FAB now opens a proper inline dialog with a name field and a directory picker (defaults to `__today`).

Settings is a full screen reached from the gear icon: repository URL, branch, personal access token (masked), plus *Test connection* and *Pull now* actions and a last-pull timestamp. Cancel discards, Save commits the config and returns to the tree with a confirmation toast.
